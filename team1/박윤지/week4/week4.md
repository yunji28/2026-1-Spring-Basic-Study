# [섹션 6]
### 1. 자바로 개발하면서 가장 불편했던 점은?
- 자바는 객체 지향 언어
- 따라서 객체의 라이프 사이클 (생성, 사용 등) 관리와 객체들 간의 의존 관계를 처리해야 했다.


### 2. 스프링의 주요 원리
1. **IoC (Inversion of Control)**
    - inverse : 정반대로 뒤집힌다
    - control : 객체에 대한 제어권 (프로그램의 흐름 = 객체의 흐름)  
    → 객체를 제어하는 주체가 정반대로 뒤집는다. ( 객체에 대한 제어권이 개발자 → **스프링 프레임 워크**로 넘어가게 되었다 )

2. **(스프링 | IoC | DI) 컨테이너**
    - 스프링이 객체를 생성해서 관리를 하기 위해 객체들을 한 곳에 담아놓은 박스

3. **스프링빈**
    - 스프링이 컨테이너에 담아두고 관리하는 객체


4. **DI (Dependency Injection)**
    - 개발자가 직접 의존 객체를 만들거나 찾는 대신, 컨테이너에게 필요한 객체를 달라고 요청하고, 이를 스프링이 제공(주입)해주는 방식
    - IoC를 가능하게 하는 핵심 메커니즘 중 하나이다
    - 객체를 사용한다 = 객체를 생성해서, 메서드/필드를 호출, 접근한다
